# 变量提升（Hoisting）

## 什么是变量提升？

JS 引擎在执行代码前有一个**编译阶段**，会把变量声明和函数声明"提升"到当前作用域的顶部。

```js
console.log(a)  // undefined（不报错）
var a = 1

// 实际执行顺序相当于：
var a           // 声明提升到顶部，值为 undefined
console.log(a)  // undefined
a = 1           // 赋值留在原地
```

---

## var / let / const 提升的区别

| | 是否提升 | 初始值 | 能否提前使用 |
|---|---|---|---|
| `var` | ✅ 提升 | `undefined` | ✅ 能（值为 undefined） |
| `let` | ✅ 提升 | 未初始化 | ❌ 暂时性死区，报错 |
| `const` | ✅ 提升 | 未初始化 | ❌ 暂时性死区，报错 |

> `let/const` 也会提升，但进入**暂时性死区（TDZ）**，声明前访问会抛出 `ReferenceError`。

```js
console.log(b)  // ReferenceError: Cannot access 'b' before initialization
let b = 2
```

---

## 函数提升

函数声明**整体提升**（包括函数体），函数表达式只提升变量声明。

```js
// 函数声明：整体提升，可以在声明前调用
sayHi()  // ✅ 'Hi'
function sayHi() { console.log('Hi') }

// 函数表达式：只提升变量，调用时还是 undefined
sayBye()  // ❌ TypeError: sayBye is not a function
var sayBye = function() { console.log('Bye') }
```

---

## 函数提升优先级高于变量提升

同名的函数声明和变量声明同时存在时，**函数声明优先**：

```js
console.log(foo)  // [Function: foo]，不是 undefined

var foo = 'bar'
function foo() {}

// 编译阶段实际顺序：
// 1. function foo(){} 提升（函数优先）
// 2. var foo（重复声明，忽略）
// 3. console.log(foo) → [Function: foo]
// 4. foo = 'bar'（赋值）
```

## 常见面试题

**题 1：输出什么？**
```js
var a = 1
function fn() {
  console.log(a)  // ?
  var a = 2
}
fn()
```
<details>
<summary>答案</summary>

`undefined`。函数内部的 `var a` 提升到函数作用域顶部，遮蔽了外部的 `a`，但赋值还没执行。

</details>

---

**题 2：输出什么？**
```js
console.log(foo)  // ?
var foo = 1
console.log(foo)  // ?
function foo() {}
console.log(foo)  // ?
```
<details>
<summary>答案</summary>

```
[Function: foo]   // 函数声明优先提升
1                 // var foo = 1 赋值执行
1                 // 函数声明不会再覆盖赋值
```

</details>

---

**题 3：输出什么？**
```js
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0)
}
```
<details>
<summary>答案</summary>

输出三次 `3`。`var` 没有块级作用域，i 是全局变量，setTimeout 回调执行时循环已结束，i 已经是 3。

**修复方案：**
```js
// 方案1：用 let（块级作用域，每次循环独立的 i）
for (let i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0)  // 0 1 2
}

// 方案2：用 IIFE 形成闭包保存当前 i
for (var i = 0; i < 3; i++) {
  ((j) => setTimeout(() => console.log(j), 0))(i)  // 0 1 2
}
```

</details>

---

**题 4：输出什么？**
```js
function fn() {
  console.log(a)  // ?
  console.log(foo())  // ?

  var a = 1
  function foo() { return 2 }
}
fn()
```
<details>
<summary>答案</summary>

```
undefined    // var a 提升但未赋值
2            // 函数声明整体提升，可以正常调用
```

</details>

---

**题 5：let 的暂时性死区**
```js
var x = 'global'
function fn() {
  console.log(x)  // ?
  let x = 'local'
}
fn()
```
<details>
<summary>答案</summary>

`ReferenceError`。函数内 `let x` 也会提升，在声明前形成暂时性死区，即使外部有同名变量也无法访问。

</details>

---

## 一句话总结

> `var` 提升且初始化为 `undefined`，可以提前用；`let/const` 提升但不初始化，提前用报错（TDZ）；函数声明整体提升，优先级高于 `var`。
