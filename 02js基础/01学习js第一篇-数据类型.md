#### 数据类型的存储位置

**基本类型** 存储在**栈内存（Stack）**，**引用类型（Object）** 存储在**堆内存（Heap）**，栈中只保存指向堆的指针。

```
栈内存                    堆内存
┌─────────────┐          ┌─────────────────────┐
│ a = 1       │          │ { name: 'foo' }      │
│ b = 'hello' │          │ [1, 2, 3]            │
│ c = true    │          │                      │
│ obj → ──────┼─────────▶│ 0x001 {...}          │
└─────────────┘          └─────────────────────┘
```

**为什么这样设计？**

| 对比项 | 栈 | 堆 |
|--------|----|----|
| 大小 | 固定，编译期可知 | 动态，运行时才确定 |
| 分配方式 | 自动分配/释放（LIFO） | 手动申请，GC 回收 |
| 访问速度 | 快 | 相对慢 |
| 存放内容 | 基本类型值、引用地址 | 对象、数组、函数 |

基本类型（number、string、boolean、null、undefined、symbol、bigint）大小固定，放栈里分配/释放开销极小；对象的大小在运行时才能确定，且可能很大，放堆里通过 GC 统一管理。

**由此引申的面试考点**

1. **赋值行为不同**：基本类型赋值是值拷贝，互不影响；引用类型赋值是拷贝指针，两个变量指向同一个堆对象，修改会相互影响。

```js
// 基本类型 —— 值拷贝
let a = 1
let b = a
b = 2
console.log(a) // 1，不受影响

// 引用类型 —— 指针拷贝
let obj1 = { x: 1 }
let obj2 = obj1
obj2.x = 99
console.log(obj1.x) // 99，受影响
```

2. **深拷贝 vs 浅拷贝** 的本质就是：是否在堆里新建一块内存来存放对象。

3. **闭包为什么不会被 GC 回收**：闭包引用的变量所在的堆内存有引用计数，只要外部还持有函数引用，GC 就不会释放它。

#### object

这是一个复杂数值类型

object   的每个实例都具有以下属性和方法

constructor 保存着创建当前对象的函数

hasOwnProprty(proprtyName)  检测proprtyName在当前对象的实例中

isProtoTypeOf(object)  用于检测传入对象是否是当前对象的原型

propertyIsEnumerable(proprtyName)  用于检查给定属性是否能够使用for-in 语句来枚举 与hasOwnProperty()   方法一样  作为参数的属性名必须以字符串形式指定
