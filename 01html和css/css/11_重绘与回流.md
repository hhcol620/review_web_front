# 重绘与回流

## 浏览器渲染流程

```
HTML → DOM树
CSS  → CSSOM树
        ↓
     Render Tree（渲染树）
        ↓
     Layout（回流）— 计算位置和尺寸
        ↓
     Paint（重绘）— 填充像素
        ↓
     Composite（合成）— 图层合并显示
```

---

## 回流（Reflow）

**定义**：元素的**几何属性**（位置、尺寸）发生变化，浏览器需要重新计算布局。

**触发条件：**

| 类型 | 示例 |
|------|------|
| 尺寸改变 | `width`、`height`、`padding`、`margin`、`border` |
| 位置改变 | `top`、`left`、`position` |
| 内容变化 | 文字数量、图片大小 |
| DOM 操作 | 添加/删除节点、`display: none` |
| 读取布局信息 | `offsetWidth`、`scrollTop`、`getComputedStyle()` |
| 窗口变化 | `resize`、改变字体大小 |

> 回流一定引起重绘，代价较高。

---

## 重绘（Repaint）

**定义**：元素的**外观属性**改变（不影响布局），浏览器只需重新绘制该元素。

**触发条件：**`color`、`background`、`visibility`、`box-shadow`、`border-radius` 等视觉属性变化。

> 重绘不一定引起回流，代价相对较低。

---

## 性能优化策略

### 1. 批量修改样式

```js
// ❌ 每次修改都触发回流
el.style.width = '100px';
el.style.height = '200px';
el.style.margin = '10px';

// ✅ 一次性修改 class
el.className = 'new-style';

// ✅ 或用 cssText
el.style.cssText = 'width:100px; height:200px; margin:10px';
```

### 2. 读写分离，避免强制同步回流

```js
// ❌ 读写交替，每次读都强制触发回流
const w1 = el1.offsetWidth;
el1.style.width = w1 + 10 + 'px';
const w2 = el2.offsetWidth; // 强制回流
el2.style.width = w2 + 10 + 'px';

// ✅ 先全部读，再全部写
const w1 = el1.offsetWidth;
const w2 = el2.offsetWidth;
el1.style.width = w1 + 10 + 'px';
el2.style.width = w2 + 10 + 'px';
```

### 3. 对频繁动画的元素脱离文档流

```css
/* 触发 GPU 合成层，跳过 Layout 和 Paint */
.animated {
  position: absolute; /* 或 fixed */
  will-change: transform;
  transform: translateZ(0); /* 强制提升为合成层 */
}
```

### 4. 使用 `transform` 和 `opacity` 做动画

这两个属性的变化只触发**合成**，不触发回流和重绘，性能最优。

```css
/* ✅ 只触发合成 */
.box { transition: transform 0.3s, opacity 0.3s; }

/* ❌ 触发回流 */
.box { transition: width 0.3s, left 0.3s; }
```

### 5. 批量 DOM 操作使用 DocumentFragment

```js
const fragment = document.createDocumentFragment();
for (let i = 0; i < 100; i++) {
  const li = document.createElement('li');
  li.textContent = i;
  fragment.appendChild(li);
}
ul.appendChild(fragment); // 只触发一次回流
```

---

## 总结

| | 回流 | 重绘 |
|--|------|------|
| 触发原因 | 几何属性变化 | 外观属性变化 |
| 影响范围 | 重新布局 + 重绘 | 仅重新绘制 |
| 性能代价 | 高 | 中 |
| 优化方向 | 减少触发次数，批量操作 | 用 transform/opacity 替代 |
