# CSS 定位流

## 定位类型总览

| `position` 值 | 名称 | 是否脱标 | 参考点 |
|--------------|------|---------|--------|
| `static` | 静态定位（默认） | 否 | 标准流 |
| `relative` | 相对定位 | 否 | 自身原来位置 |
| `absolute` | 绝对定位 | 是 | 最近定位祖先元素 |
| `fixed` | 固定定位 | 是 | 视口（viewport） |

---

## 相对定位 `relative`

- 相对于**自身在标准流中的原始位置**移动。
- **不脱离标准流**，原来的位置仍然占用。
- 仍然区分块级 / 行内 / 行内块元素。
- 设置 `margin`/`padding` 作用于标准流中的原始位置，不影响定位后的实际位置。
- 同一方向只能用一个定位属性（如 `left` 和 `right` 不能同时生效）。

**使用场景：**
1. 微调元素位置。
2. 为绝对定位子元素提供参考点（"子绝父相"）。

---

## 绝对定位 `absolute`

- **脱离标准流**，不占用原来位置。
- 不区分块级 / 行内 / 行内块元素（类似浮动）。

### 参考点规则

1. 默认以 `body` 为参考点（以**首屏**宽高为基准，不是整个页面）。
2. 若有设置了定位的祖先元素（`relative` / `absolute` / `fixed`，不含 `static`），则以**最近的**定位祖先为参考点。
3. 绝对定位元素会忽略祖先元素的 `padding`，直接贴在 `border` 内侧。

### 常用布局：子绝父相

```css
.parent { position: relative; }
.child  { position: absolute; top: 0; right: 0; }
```

---

## 固定定位 `fixed`

- **脱离标准流**，相对于**浏览器视口**定位，滚动页面时位置不变。
- 不区分元素类型。

```css
.fixed-header {
  position: fixed;
  top: 0;
  left: 0;
  width: 100%;
}
```

---

## z-index 层叠顺序

控制定位元素之间的覆盖关系（仅对定位流元素有效）。

| 规则 | 说明 |
|------|------|
| 默认值 | 所有元素 `z-index: 0` |
| 定位 vs 标准流 | 定位流元素默认覆盖标准流元素 |
| 相同层级 | 后写的覆盖先写的 |
| 不同 z-index | 值越大越显示在上层 |

### 从父现象

```
父元素没有设置 z-index → 子元素的 z-index 直接比较
父元素设置了 z-index  → 子元素的 z-index 失效，以父元素的 z-index 为准
```

```css
/* 示例：子元素层叠由父元素决定 */
.parent-a { position: relative; z-index: 1; }
.parent-b { position: relative; z-index: 2; }
/* 无论 .child-a 的 z-index 多大，都会被 .parent-b 及其子元素压住 */
```
