# CSS 浮动流

## 浮动流的特点

- 不区分块级 / 行内 / 行内块元素，所有元素都可以**水平排列**。
- 浮动元素都可以设置宽高（类似行内块）。
- 浮动流中**没有居中对齐**，`margin: 0 auto` 无效。

---

## 浮动元素脱标

元素设置 `float` 后脱离标准流，原来的位置不再占用，后面未浮动的元素会"顶上来"被覆盖。

### 排列规则

| 情况 | 规则 |
|------|------|
| 相同方向浮动 | 先浮动的在前，后浮动的在后 |
| 不同方向浮动 | 左浮动找左浮动，右浮动找右浮动 |
| 浮动后的位置 | 由元素在标准流中的原始位置决定 |

### 字围现象

左侧元素设置左浮动后，同级的文字内容会自动环绕在浮动元素周围（常用于图文混排）。

---

## 清除浮动

浮动子元素会导致父元素高度塌陷，需要清除浮动。

### 方法一：给父元素设置固定高度

```css
.parent { height: 200px; }
```

缺点：高度固定，不灵活。

### 方法二：`clear` 属性

给浮动元素**后面**的兄弟元素设置：

```css
.next-sibling { clear: both; }
/* clear: left  — 不受前面左浮动影响 */
/* clear: right — 不受前面右浮动影响 */
/* clear: both  — 不受前面任何浮动影响 */
```

> 注意：设置 `clear` 后，该元素的 `margin-top` 会失效。

### 方法三：隔墙法

**外墙法**：在两个浮动盒子之间插入一个空块级元素并设置 `clear: both`。
- 可让第二个盒子使用 `margin-top`，但无法让第一个盒子使用 `margin-bottom`，且无法撑起父盒子高度。

**内墙法**：在第一个浮动盒子的**内部末尾**插入空块级元素并设置 `clear: both`。
- 可以撑起父盒子高度，两个盒子都可以正常使用 `margin`。

实际开发中隔墙法较少用（需要添加无意义的 DOM 元素）。

### 方法四：伪元素（推荐）

用伪元素模拟内墙法，不污染 HTML 结构：

```css
.clearfix::after {
  content: '';
  display: block;
  clear: both;
}
/* 兼容 IE6 */
.clearfix { *zoom: 1; }
```

### 方法五：`overflow: hidden`（推荐）

触发 BFC，自动包裹浮动子元素：

```css
.parent { overflow: hidden; }
/* 兼容 IE6 */
.parent { *zoom: 1; }
```

`overflow: hidden` 的额外作用：
- 裁剪超出容器的内容。
- 防止子元素 `margin-top` 穿透父元素（解决外边距折叠）。
