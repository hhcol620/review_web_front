# 前端性能优化指标

## 一、Core Web Vitals（谷歌核心指标）

Google 官方的三个核心指标，直接影响 SEO 排名：

| 指标 | 全称 | 衡量内容 | 良好标准 |
|---|---|---|---|
| **LCP** | Largest Contentful Paint | 最大内容渲染时间（用户看到主要内容的时间） | ≤ 2.5s |
| **INP** | Interaction to Next Paint | 交互响应延迟（点击/键盘/触摸到页面响应） | ≤ 200ms |
| **CLS** | Cumulative Layout Shift | 累计布局偏移（页面内容意外移动的程度） | ≤ 0.1 |

> INP 于 2024 年替代了 FID（First Input Delay）

---

## 二、其他关键指标

| 指标 | 全称 | 含义 | 良好标准 |
|---|---|---|---|
| **FCP** | First Contentful Paint | 首次内容渲染（第一个文字/图片出现） | ≤ 1.8s |
| **TTFB** | Time to First Byte | 首字节时间（发请求到收到第一个字节） | ≤ 800ms |
| **TTI** | Time to Interactive | 可交互时间（页面完全可响应用户操作） | ≤ 3.8s |
| **TBT** | Total Blocking Time | 总阻塞时间（FCP 到 TTI 之间主线程阻塞总时长） | ≤ 200ms |
| **SI** | Speed Index | 速度指数（视觉填充速度） | ≤ 3.4s |

---

## 三、指标之间的关系

```
导航开始
  │
  ├─ TTFB         服务器响应快不快
  │
  ├─ FCP          第一个内容出现（白屏结束）
  │
  ├─ LCP          最大内容出现（主体内容加载完）
  │
  ├─── TBT ───┤   主线程被长任务阻塞的总时间
  │
  └─ TTI          用户可以正常交互了

CLS 贯穿整个生命周期，衡量布局稳定性
INP 衡量整个访问期间的交互延迟
```

## 四、如何计算 / 采集

### 方式 1：Performance API（代码采集）

```js
// 获取 FCP
const observer = new PerformanceObserver((list) => {
  for (const entry of list.getEntries()) {
    if (entry.name === 'first-contentful-paint') {
      console.log('FCP:', entry.startTime)
    }
  }
})
observer.observe({ type: 'paint', buffered: true })

// 获取 LCP
const lcpObserver = new PerformanceObserver((list) => {
  const entries = list.getEntries()
  const lcp = entries[entries.length - 1]  // 取最后一个（最新的最大内容）
  console.log('LCP:', lcp.startTime)
})
lcpObserver.observe({ type: 'largest-contentful-paint', buffered: true })

// 获取 CLS
let clsValue = 0
const clsObserver = new PerformanceObserver((list) => {
  for (const entry of list.getEntries()) {
    if (!entry.hadRecentInput) {  // 排除用户交互引起的偏移
      clsValue += entry.value
    }
  }
  console.log('CLS:', clsValue)
})
clsObserver.observe({ type: 'layout-shift', buffered: true })

// 获取 TTFB
const [navEntry] = performance.getEntriesByType('navigation')
console.log('TTFB:', navEntry.responseStart - navEntry.requestStart)
```

### 方式 2：web-vitals 库（推荐，一行搞定）

```js
import { onLCP, onINP, onCLS, onFCP, onTTFB } from 'web-vitals'

onLCP(({ value }) => console.log('LCP:', value))
onINP(({ value }) => console.log('INP:', value))
onCLS(({ value }) => console.log('CLS:', value))
onFCP(({ value }) => console.log('FCP:', value))
onTTFB(({ value }) => console.log('TTFB:', value))

// 上报到后端
onLCP(({ value, id, name }) => {
  fetch('/analytics', {
    method: 'POST',
    body: JSON.stringify({ name, value, id })
  })
})
```

### 方式 3：工具测量

```
Lighthouse      → Chrome DevTools 内置，模拟测量，给出分数和建议
PageSpeed Insights → Google 在线工具，真实用户数据 + 实验室数据
Chrome DevTools Performance 面板 → 录制详细火焰图，排查长任务
Web Vitals 扩展 → Chrome 插件，实时显示当前页面指标
```

## 五、各指标优化方向

### LCP 优化（最大内容渲染）
```
1. 对 LCP 元素（通常是首屏大图/Hero区域）加 fetchpriority="high"
   <img src="hero.jpg" fetchpriority="high" />

2. 预加载关键资源
   <link rel="preload" as="image" href="hero.jpg" />

3. 使用 CDN，减少资源加载时间

4. 图片格式优化：WebP/AVIF 体积更小

5. 服务端渲染（SSR）让 HTML 直接包含内容，不依赖 JS 执行
```

### CLS 优化（布局稳定性）
```
1. 图片/视频设置固定宽高，避免加载后撑开页面
   <img width="800" height="400" src="..." />

2. 广告位预留固定空间

3. 避免在已有内容上方动态插入内容

4. 字体使用 font-display: optional 避免字体切换导致位移
```

### INP 优化（交互响应）
```
1. 避免长任务（> 50ms 的任务）阻塞主线程
2. 用 Web Worker 把计算密集型任务移到后台线程
3. 事件处理函数保持轻量，复杂操作用 setTimeout 拆分
4. 减少不必要的 re-render（React memo / Vue computed）
```

### TTFB 优化（首字节时间）
```
1. 使用 CDN，让用户就近访问
2. 服务端开启 HTTP 缓存（Cache-Control）
3. 数据库查询优化，减少服务器处理时间
4. 使用 HTTP/2 多路复用
```

### FCP 优化（首次内容渲染）
```
1. 减少渲染阻塞资源：CSS 内联关键样式，JS 加 defer/async
2. 服务端渲染（SSR）/ 静态生成（SSG）
3. 减小 HTML 体积
```

## 六、面试常考题

**Q: 白屏时间怎么计算？**
```js
// 方式1：Performance API
const whiteScreenTime = performance.timing.domInteractive
  - performance.timing.navigationStart

// 方式2：FCP 近似表示白屏结束时间
// FCP 就是"第一个内容出现"的时间点
```

**Q: 首屏时间怎么计算？**
```js
// 手动打点：在首屏内容渲染完成后记录时间
const firstScreenTime = Date.now() - performance.timing.navigationStart

// 或监听 LCP，LCP 通常代表首屏主要内容加载完毕
```

**Q: Core Web Vitals 不达标怎么排查？**
```
LCP 差 → 检查首屏最大元素是什么，是否有预加载，图片是否过大
CLS 差 → 检查是否有无尺寸图片、动态插入内容、字体闪烁
INP 差 → Performance 面板录制交互，找长任务，检查事件处理函数耗时
```
